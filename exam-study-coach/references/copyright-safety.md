# Copyright and Safety Boundaries

Use this reference before summarizing materials, quoting sources, searching the web, or preparing public output.

## Core Rule

Help the user learn from materials they provide. Do not redistribute those materials.

## Allowed

- Summarize user-provided materials in short paraphrases.
- Extract topic names, formulas, concepts, and study priorities.
- Create original practice questions.
- Link to public official pages, open course pages, open textbooks, public videos, and documentation.
- Quote brief fragments only when needed for precision.
- Produce private study artifacts in the user's workspace.

## Avoid

- Uploading or publishing teacher slides, textbook scans, full exam papers, answer keys, or classmates' work.
- Reproducing large passages from copyrighted materials.
- Creating a public repository containing course files or private school materials.
- Revealing personal data from notes, screenshots, class rosters, grades, student IDs, or chat logs.
- Claiming public sharing is legal just because the material is educational.
- Publishing real past exam papers, teacher-marked focus notes, or original answer keys from a private course.
- Treating past exams as a guarantee of future questions.

## Web Search

Search the web only when the user asks for public resources or explicitly permits it. Prefer official or clearly public sources. Cite links in the output.

## Public GitHub Output

If creating a public GitHub repository, include only:

- Skill instructions.
- Scripts.
- Empty templates.
- Safety guidance.
- Synthetic examples.

Do not include real course materials or real student answer records.

## Past Exams and Teacher Focus

Use `past-exams/`, `past-exam-answers/`, and `teacher-focus/` only for private study guidance unless the user confirms the material is public and authorized for redistribution.

Prefer generating original similar questions from the tested concept and style. If the user asks for original past-paper practice, keep it private and avoid copying large passages into public outputs.

## If Risk Appears

If the user asks to publish questionable material:

1. State the risk briefly.
2. Offer a safer alternative: summary, checklist, original practice questions, or private-only local study pack.
3. Continue with the safe version.
